# RTF Strata Thickness And Material Weighting Plan

## Objective

Make RTF strata configurable enough to avoid thin noisy bands and excessive non-stone underground
material while preserving the strata system. The implementation must affect real generated surface
rules and exported datapacks, not just GUI state.

## Current Behavior

- `strataDecorator` enables the whole strata rule.
- `strataRegionSize` controls horizontal X/Z scale for strata region selection and depth noise. It
  is not a vertical layer-count or band-thickness setting.
- Vertical rock layer count and rock material weights are now configurable through
  `MiscellaneousSettings.StrataSettings`.
- Rock strata are currently built from explicit weighted materials: `stone`, `granite`, `andesite`,
  and `diorite`.
- Current defaults are `rockMinLayers = 4`, `rockMaxLayers = 10`, `rockMinDepth = 0.1`,
  `rockMaxDepth = 1.5`, and `stoneWeight = 1.5`.

## Current QA Finding

Client QA on a `worldDepth` around `128` found that the initial configurable strata fix improved the
intended surface/near-surface layers: bands are thicker and stone is more common because of
weighting. However, deepslate still starts around `Y=0` to `Y=-10` while the world extends down near
`Y=-120`.

That leaves too much of the configured underground range as deepslate. This is not caused by the
tall-mountain height projection. It appears to be a remaining vanilla-band anchoring issue: the
generated surface rules still include `SurfaceRuleData.overworld()`, whose vanilla deepslate
transition is tied to the normal Minecraft Y band rather than being stretched or remapped to the
configured `worldDepth`.

Expected follow-up behavior:

- keep the improved configurable strata layer thickness and stone weighting
- make the deepslate transition respect configured depth, or add a preset-owned deepslate transition
  band
- for a world bottom near `Y=-128`, avoid starting deepslate immediately around `Y=0` unless the
  preset explicitly asks for that

## Original Direct Answer

Decreasing the number of vertical strata layers was not originally configurable. `strataRegionSize`
was not the correct knob for this; it changes horizontal region size and noise scale.

## Proposed Preset Options

Add a nested strata configuration to `MiscellaneousSettings`, keeping defaults compatible with
current behavior unless explicitly changed.

Suggested shape:

```json
"strata": {
  "rockMinLayers": 4,
  "rockMaxLayers": 10,
  "rockMinDepth": 0.1,
  "rockMaxDepth": 1.5,
  "stoneWeight": 1.5,
  "graniteWeight": 1.0,
  "andesiteWeight": 1.0,
  "dioriteWeight": 1.0
}
```

Recommended initial defaults:

- `rockMinLayers = 4`
- `rockMaxLayers = 10`
- `stoneWeight = 1.5`
- `graniteWeight = 1.0`
- `andesiteWeight = 1.0`
- `dioriteWeight = 1.0`

Rationale: fewer rock layers create thicker bands; stone weighting makes plain stone bands more
common without deleting alternate rock types.

## Implementation Plan

1. Add preset schema.
   - Add a `StrataSettings` nested class or record under `MiscellaneousSettings`.
   - Decode it with `optionalFieldOf("strata", StrataSettings.DEFAULT)`.
   - Include it in `copy()`.
   - Keep old presets loadable when the new object is absent.

2. Wire layer count into actual surface generation.
   - Change `PresetSurfaceRuleData.makeStrataRule` to accept the full `Preset` or
     `MiscellaneousSettings`.
   - Replace hardcoded rock `new Strata(..., 10, 30, 0.1F, 1.5F)` with values from
     `miscellaneous.strata`.
   - Keep soil/sediment/clay hardcoded initially unless a broader strata config is desired.

3. Add weighted rock material selection.
   - Extend `StrataRule.Strata` to support optional material weights.
   - Do not rely on tags alone for weighting; tags are unweighted.
   - For the initial implementation, add a special weighted rock path generated by
     `PresetSurfaceRuleData` using explicit block ids and weights.
   - Ensure generated datapack JSON includes weight data in the `reterraforged:strata` rule.

4. Preserve default behavior intentionally.
   - If QA expects the new default to be improved, set defaults to fewer layers and stone weight
     `1.5`.
   - If upstream compatibility is more important, preserve current `10..30` layers by default and
     only use the new values when explicitly present. Decide before implementation.

5. Add GUI controls.
   - Add sliders on `MiscellaneousPage`:
     - Rock Min Layers
     - Rock Max Layers
     - Stone Strata Weight
   - Optional follow-up sliders:
     - Granite Weight
     - Andesite Weight
     - Diorite Weight
   - Clamp `rockMaxLayers >= rockMinLayers`.

6. Export and QA.
   - Export a datapack with default settings and inspect
     `data/minecraft/worldgen/noise_settings/overworld.json`.
   - Confirm the `reterraforged:strata` rule contains the configured layer counts and weights.
   - Generate the same seed twice with different `rockMinLayers/rockMaxLayers`; band thickness must
     visibly change.
   - Generate the same seed twice with different `stoneWeight`; stone frequency must visibly change.

## QA Acceptance Criteria

- `strataRegionSize` remains horizontal-scale behavior and is not mislabeled as vertical thickness.
- `rockMinLayers` and `rockMaxLayers` change the number and thickness of vertical strata bands.
- `stoneWeight = 1.5` makes stone bands more common than individual granite/andesite/diorite bands.
- Increasing `stoneWeight` further produces visibly more stone without disabling strata.
- Deepslate transition depth respects configured `worldDepth` or an explicit preset setting, rather
  than remaining anchored around vanilla `Y=0`.
- Existing presets without the new `strata` object still load.
- Exported datapacks preserve the configured values.

## Notes For Later Agent

- The current `StrataRule.Strata` codec only supports tag-based material pools. Weighted material
  selection will require a codec change or a new strata entry type.
- A simple first version can keep non-rock strata unchanged and only make rock strata
  explicit/weighted.
- Avoid solving this by editing `#reterraforged:rock` alone. That can remove materials, but it
  cannot express weighted stone preference or thicker bands.
